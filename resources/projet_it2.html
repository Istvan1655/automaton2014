Projet Informatique théorique 2

Le projet est à réaliser par équipes de 3 personnes maximum. Il comporte une partie à rédiger sur papier, et un programme.

Dates de remise (attention, dates strictes) :

  • Vendredi 9 mai 2014, 12h00 pour la partie à rédiger,
  • Mercredi 14 mai 2014, 12h00 pour le programme.

Contrairement au projet de l'UE AS&PP3, il n'y aura pas de soutenance et on ne demande pas de rapport, mais le code doit être bien commenté. La notation tiendra compte du degré de réalisation du sujet, de la qualité du code (portabilité, lisibilité, documentation), et du jeu de tests.

Les réponses à la partie à rédiger (lien) sont à rendre au format pdf, de préférence produit par LaTeX, par mail à l'adresse it2@labri.fr.

La programme est à rendre au format archive tar compressée par gzip, sous le nom nom1-nom2-nom3.tgz par mail à l'adresse it2@labri.fr. La décompression de l'archive doit fournir un répertoire nom1-nom2-nom3 dans lequel se trouveront les fichiers.

La partie programmation consiste à compléter le fichier automate.c fourni dans l'archive projet-2014.tgz. L'interface automate.h contient les prototypes des fonctions, dont celles à implémenter. Le code est commenté. Des tests sont fournis dans le fichier evaluation.c. Les seuls fichiers à modifier sont les fichiers automate.c et evaluation.c (les autres seront remplacés par les fichiers originaux lors des tests). Vous devez également utiliser le fichier makefile fourni. Il produit un exécutable evaluation qui lance les tests et affiche le nombre de tests réussis et le nombre de tests échoués. Par défaut, les tests sont lancés avec un automate vide. Il est conseillé d'ajouter des tests par vous-mêmes car nous utiliserons un fichier plus complet lors de l'évaluation.
